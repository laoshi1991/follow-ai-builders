# 项目结构文档（follow-ai-builders）

本项目提供一个中心化的内容聚合与摘要体系，用于产出 “AI Builders Digest”。核心由三部分组成：
- 数据抓取与去重（CI 定时运行，生成 feed JSON）
- 本地准备摘要输入（聚合远端 feed、加载 prompts 与用户配置）
- 摘要投递（Telegram、Email 或终端）

本文梳理目录结构、关键脚本、配置与数据流，以及如何本地调试或扩展。

## 数据流与职责

- 生成阶段（CI）
  - 定时任务在 GitHub Actions 上运行 Node 20，调用 `scripts/generate-feed.js`：
    - X/Twitter：官方 API v2，按 24 小时回溯窗口拉取 25 位建造者推文，按用户最多 3 条，去重写入 `state-feed.json`
    - 播客（YouTube）：通过 Supadata 抓取频道/播放列表视频，挑选 72 小时窗口内最早未见的视频，并拉取转写文本
    - 官方博客：抓取 Anthropic Engineering 与 Claude Blog 的索引页，解析最新条目并抓取正文内容
  - 三类产物写入仓库根目录：
    - `feed-x.json`、`feed-podcasts.json`、`feed-blogs.json`
  - `state-feed.json` 用于跨次运行去重（保留 7 天窗口后清理）

- 准备阶段（本地/Agent）
  - 运行 `scripts/prepare-digest.js`：
    - 从仓库公开原始地址获取三类 feed（raw.githubusercontent.com）
    - 读取用户配置 `~/.follow-builders/config.json`
    - 加载 prompts（优先级：用户自定义 > GitHub 最新 > 本地默认）
    - 汇总为单一 JSON 输出（stdout），供 LLM 严格按提示词进行“重混与撰写”

- 投递阶段（可选）
  - 将 LLM 生成的纯文本摘要输入 `scripts/deliver.js`：
    - `stdout`（默认）：直接打印
    - `telegram`：需要 `~/.follow-builders/.env` 内的 `TELEGRAM_BOT_TOKEN` 与配置中的 `chatId`
    - `email`（Resend）：需要 `RESEND_API_KEY` 与收件地址

## 核心脚本说明

- generate-feed.js（CI 使用）
  - 输入：`config/default-sources.json`
  - 机密：`X_BEARER_TOKEN`、`SUPADATA_API_KEY`（通过 GitHub Secrets 注入）
  - 输出：`feed-x.json`、`feed-podcasts.json`、`feed-blogs.json`、更新去重状态 `state-feed.json`
  - 特性：严格去重、时间窗口过滤、稳健的解析/回退策略（例如 Next.js __NEXT_DATA__ 与正则解析）

- prepare-digest.js（本地/Agent）
  - 拉取三类 feed、读取用户偏好、收集 prompts，输出供 LLM 使用的一体化 JSON
  - 用户配置路径：`~/.follow-builders/config.json`（结构参考下文 Schema）
  - 远端 prompts 基于本仓库 `prompts/` 最新版本；用户可在本地 `~/.follow-builders/prompts/` 覆盖

- deliver.js（本地/Agent）
  - 输入：摘要文本（stdin、`--message` 或 `--file`）
  - 方式：
    - stdout：直接输出
    - Telegram：分片发送（处理 4096 字符限制），失败回退至无 Markdown 模式
    - Email（Resend）：主题包含日期美化

## CI 工作流

- 定时触发：每日 06:00 UTC
- 手工触发：workflow_dispatch（支持 `all`、`tweets-only`、`podcasts-only`、`blogs-only`）
- 步骤摘要：
  1) Checkout → Node 20
  2) `cd scripts && npm install`
  3) 执行 `node generate-feed.js`（按输入 mode 选择）
  4) 将 feed 与状态文件提交回仓库

工作流文件：[.github/workflows/generate-feed.yml](file:///Users/caolaoshi/skills/follow-ai-builders/.github/workflows/generate-feed.yml)

## 本地调试与运行

- 准备依赖

```bash
cd scripts && npm install
```

- 本地运行 prepare（无需任何 API Key）

```bash
cd scripts && node prepare-digest.js
```

运行后在 stdout 输出一个包含 `config`、`podcasts`、`x`、`blogs`、`prompts`、`stats` 的 JSON。

- 本地运行 deliver

```bash
echo "digest text" | node scripts/deliver.js
# 或
node scripts/deliver.js --message "digest text"
```

- 如需本地生成 feed（一般只在 CI 中运行）
  - 需要设置环境变量：`X_BEARER_TOKEN`、`SUPADATA_API_KEY`

```bash
cd scripts && X_BEARER_TOKEN=xxx SUPADATA_API_KEY=yyy node generate-feed.js
```

## 扩展与自定义

- Prompt 自定义
  - 建议通过对话调整；或复制到本地覆盖目录：
  - `~/.follow-builders/prompts/` 同名文件将优先于远端与仓库版本

- 信息源更新
  - 源清单由仓库集中维护，默认由 CI 使用 `config/default-sources.json` 拉取
  - 如需增减源，请在该文件中修改并配合 CI 秘钥运行；或在上游仓库提交 PR
